GIFT RapidMiner Developer Information

In the current integration, GIFT has been integrated with RapidMiner version 5.3.013.  GIFT can use RapidMiner models inside of the GIFT Learner Module.  RapidMiner models can be created in the RapidMiner authoring tool as seen in the screenshot below (RapidMiner site).

Once created your model can be used in a GIFT Learner module classifier or predictor as an algorithm for calculating learner state.  Both classifiers and predictors exists as Java code in GIFT but can be written in such a way that the algorithm can reside in another programming language (e.g. python) or third party library (e.g. RapidMiner). 

A classifier consumes data coming into the learner module.  This data can be from sensors (e.g. ECG), learning management systems (LMS), learner record store (LRS), real time performance assessments (from the GIFT Domain module), GIFT survey responses and training application / game state information.  Classifiers analyze this data and output short-term learner state values (e.g. Low Anxious).  On the other hand a predictor consumes the same data set as well as the classifier's output to create predicted learner state values. 

To specify this data pipeline that runs during course execution in the learner module (i.e. what data goes into the classifier/predictor) you need to edit the learner configuration for your course.  This can be accomplished in the GIFT course authoring tool under "Course Properties" by editing the learner configuration. 

For more information please read GIFT RapidMiner Developer Guide.
