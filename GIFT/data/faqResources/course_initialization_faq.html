Why am I seeing the Gateway Module setup page?

The course you are currently running will require interaction with a training application (e.g. PowerPoint, VBS, DE Testbed) at some point.  These training applications cannot be run in your browser and instead must run using installed software on your computer.  Therefore in order to continue this course you must download and install the Gateway Module. 

Why do I need to download the Gateway Module?

The Gateway Module is needed in order for GIFT to be able to communicate with and sometimes control the training applications on your computer needed for this course. Due to web security restrictions, you will need to first download this application, then run it.  In some instances you maybe asked for configuration information for these training applications before the Gateway Module can begin communicating with GIFT.  Once configured, you will be allowed to continue the course.

Why is there a notification that the file may harm my computer?

Whether or not you see a notification about the GIFT Gateway Module file (.zip) depends on the security settings of your browser.  By default you should see at least one prompt notifying you that the file could be harmful - if you don't trust the creator.  If you want to execute the GIFT course you will have to accept the risk. 

I get a message "Windows Firewall has blocked some features of this program".  What do I do?

Depending on your security settings, you may get a Windows prompt stating "Windows Firewall has blocked some features of this program" when running the GIFT Gateway Module.  If you want to run the GIFT course, you will need to select "Allow access" when running the GIFT Gateway Module.   The GIFT Gateway Module communicates with the GIFT server and external training applications (such as PowerPoint, VBS, DE Testbed) and needs to be allowed access to the network. 

I'm unable to open the Gateway Module once it downloads.   What do I do?

After the Gateway Module downloads, you need to open it manually.  If you're having trouble opening the Gateway Module, please make sure that the *zip file has been extracted, you have navigated to within the extracted folder, and you have run startGIFTGateway.bat

Where does the Gateway Module get saved when it is downloaded?

The Gateway Module file (.zip) will be saved to the downloads folder for your browser. 

Why can't I run a previously downloaded version of the Gateway Module?

In order to guarantee the latest experience with the GIFT version you are interacting with you will need to use the version provided at the point when executing the course.  If you don't, not only could you experience a legacy interaction but the Gateway Module or course might not work at all.  To see the GIFT version information associated with the Gateway Module, use the system tray icon to view the "About" dialog.

I've downloaded the Gateway Module and accepted the security risk, but nothing is happening. What do I do?

If nothing is happening, then something might have gone wrong with the Gateway module's installation process. To contact support, create a forum post at the GIFT Troubleshooting Forum describing your issue and what steps led you to it.

I see the following exception when running the Gateway Module: "Caught exception while trying to establish connection to ActiveMQ. Is ActiveMQ running at tcp://gifttutoring.org:1001?" How can I resolve this?

If you see an exception that reads "Caught exception while trying to establish connection to ActiveMQ. Is ActiveMQ running at tcp://gifttutoring.org:1001?" when running the Gateway Module, please make sure that your network has tcp port 1001 open for both inbound and outbound traffic. The Gateway Module communicates via tcp port 1001 for sending and receiving messages to/from the GIFT server. If that port is not open, then that exception will be generated.
