Eliminate Hostiles Condition

This condition checks whether a list of pre-defined entities/characters in the external application have been eliminated.  In order to identify these entities their initial locations must be defined for this condition.  An entity is eliminated if their health status is anything but healthy.  The condition doesn't look at the cause of the change in health status so you can't, for example, use this condition to gaurantee that the learner was the cause of the change in health unless your environment/scenario guarantees that on its own (e.g. the learner is the only entity that will cause harm to the target entity).

Assessment Evaluations (Real-time)

     Default:

  • Above Expectation
  • At Expectation
    • case: all of the entities to track are no longer healthy
  • Below Expectation
    • case: the default value
    • case: if there is at least one healthy entity from the list of entities to track
  • Unknown

     Customize:

  • count: an evaluation event happens once for each entity in the authored list when the entity is damaged.  The number of events will be less than or equal to the number of entities.
  • completion time: the amount of time from the start of the task to when all entities in the list are damaged.  

Possible Scored Evaluations (overall)

  • counts the number of entities from the list that are no longer healthy (not to exceed the number of entities in the list)
  • provides the amount of time it took for all entities in the list to no longer be healthy

Miscellaneous details

  • the condition completes when all entities from the list are no longer healthy
  • the condition can't be repeated

Intended Training Applications and Domains

  • any application that can cause GIFT Entity state messages to be created

Consumed State Messages (used for assessment)

  • GIFT Entity State
