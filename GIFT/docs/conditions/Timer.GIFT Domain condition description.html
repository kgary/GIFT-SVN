Timer Condition

This condition is useful for sequencing assessments or instructional strategies by using the fact that this condition will result in an At Expectation assessment as well as be completed after the defined duration.  The author can also choose to have the timer repeat until the parent concept is no longer in an active task.  While configured to repeat, the condition will not end.

Assessment Evaluations (Real-time)

  • Above Expectation
  • At Expectation
    • case: everytime the timer reaches zero
  • Below Expectation
  • Unknown
    • case: the default value
    • case: everytime the timer starts again

Possible Scored Evaluations (overall)

  • count - the number of times the timer has expired.  If the condition doesn't repeat this value will be 0 or 1, for timer did not expire and timer expired respectively.
  • time - if the condition doesn't repeat the time value will be the amount of time the timer was set to by the author.

Miscellaneous details

  • the condition will complete if not authored to repeat
  • the condition can repeat
  • the condition can loop if authored to do so (by choosing to repeat the timer)

Intended Training Applications and Domains

  • any application that can produce simulation management (Siman) messages

Consumed State Messages (used for assessment)

  • GIFT Siman
