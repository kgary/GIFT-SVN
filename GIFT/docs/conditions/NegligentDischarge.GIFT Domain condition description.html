Negligent Discharge Condition

This condition assesses whether one or more team members fired their weapon towards an object.

The author can define the objects to avoid, and weapon cone as inputs to the assessment algorithm.  Only targets that are healthy will be considered for assessment.  This prevents being assessed against objects that are dead/incapacitated.

The goal of this condition is to not assess every possible situation.  Hopefully it provides an initial assessment result that can be used by observers/trainers to make additional assessments or over-rides.

Assessment Evaluations (Real-time)

  • Above Expectation
  • At Expectation
    • case: default
  • Below Expectation
    • case: [Negligent Discharge] assessed team member fires their weapon when an object to avoid is in the weapon cone.
  • Unknown

Possible Scored Evaluations (overall)

  • counts the number of violations (going from At to Below Expecation)
  • sums the duration of time in Below Expectation

Miscellaneous details

  • this condition does not complete

Intended Training Applications and Domains

  • any application that can produce entity state and weapon fire messages

Consumed State Messages (used for assessment)

  • GIFT Entity State
  • GIFT Weapon Fire
