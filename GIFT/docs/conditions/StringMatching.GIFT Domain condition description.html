String Matching Condition

This condition is used to match a single key value pair provided to the condition as input.  For example, the key maybe "opened" and the value might be "door A" to indicate that the learner opened door A in the application.  The abstract nature of this condition allows you to easily assess events communicated by your application using with minimal software development in the GIFT domain module. 

This condition is talked about in the GIFT Software Developer guide.  It is also used by multiple training applications that are already integrated with GIFT (see Training Applications section).

Assessment Evaluations (Real-time)

  • Above Expectation
  • At Expectation
    • case: when the key value pair in the condition matches the key value pair provided
  • Below Expectation
    • case: when the key value pair in the condition does NOT match the key value pair provided
  • Unknown
    • case: the default value

Possible Scored Evaluations (overall)

  • counts the number of times the key value pair matches
  • provides the amount of time it took to reach At Expectation, i.e. the key value pair matched

Miscellaneous details

  • the condition does not complete
  • the condition can be repeated

Intended Training Applications and Domains

  • Unity (w/ the GIFT Unity SDK)
  • GIFT Demo Application (a C# desktop application that is to be used with the GIFT software developer guide)
  • any application that can produce the Simple Example State message

Consumed State Messages (used for assessment)

  • GIFT Simple Example State
