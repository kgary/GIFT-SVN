Muzzle Flagging Condition

This condition checks whether the entity controlled by a player is pointing their weapon at another entity which is being assessed by this condition. The threshold in degrees for how closely a weapon is allowed to be pointed at an entity can be configured within the condition.

Optionally, the condition can also be configured to only assess entities who are within a certain range of one another and whether the assessed member's weapon safety value should be considered.

Assessment Evaluations (Real-time)

  • Above Expectation
  • At Expectation
    • case: No assessed entities are pointing their weapons closer than the configured angle threshold at any other assessed entity within the distance optional distance threshold (if authored).
  • Below Expectation
    • case: At least one assessed entity is pointing their weapon at another assessed entity within the angle threshold, weapon safety is off, and the two entities are within the distance threshold (if authored).
  • Unknown
    • case: The default initial value of this condition. This default value will remain until at least one entity state for two of the assessed entities is received.

Possible Scored Evaluations (overall)

  • counts the number of violations (going from At to Below Expecation)
  • sums the duration of time in Below Expectation

Miscellaneous details

  • this condition does not complete

Intended Training Applications and Domains

  • any application that can produce entity state messages

Consumed State Messages (used for assessment)

  • GIFT Entity State
  • Variables State Result
