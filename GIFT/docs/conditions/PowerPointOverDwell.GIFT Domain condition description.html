PowerPoint Over Dwell Condition

This condition checks whether the learner has stayed too long on a particular PowerPoint slide based on timing information provided.  The timing information can be provided as a single default number for all slide indexes or on a per slide index basis.

Assessment Evaluations (Real-time)

  • Above Expectation
  • At Expectation
    • case: as long as the learner leaves the current slide before any slide timer reaches zero
  • Below Expectation
    • case: when the current slide's timer reaches zero and the learner is still on that slide
  • Unknown
    • case: the default value

Possible Scored Evaluations (overall)

  • counts the number of timer's that reach zero when the learner is still on that slide, at most this will be equal to the number of slides reached in the presentation

Miscellaneous details

  • the condition completes when the Siman.Stop_Freeze is received which for a PowerPoint show means that the show has been closed
  • this condition can be repeated, i.e. as a slide index is reached a timer is created.  If the show has logic to jump around or repeat slides than the over dwell logic can be repeated

Intended Training Applications and Domains

  • PowerPoint slide shows

Consumed State Messages (used for assessment)

  • GIFT PowerPoint State
  • GIFT Siman.Stop_Freeze
