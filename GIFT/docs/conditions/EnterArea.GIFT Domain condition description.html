Enter Area Condition

This condition checks whether the learner has entered a location among one or more locations defined for this condition.  For this condition entering a location is defined as reaching a location (within a defined radius) called the start location followed by reaching a second but different location (within a defined radius) called the end point.  The learner can only be assessed against one of these start/end location sets at a time within this condition.  This means if a learner reaches the start point for one entrance and then reaches the start point for a second entrance, the first entrance will no longer be checked against.  This condition is useful for checking whether the learner has entered an area that you may or may not want them to be in (e.g. which doorway was entered in a building).  It is because of either use case that each entrance requires the author to define the appropriate assessment level if that entrance is crossed by the learner.

Assessment Evaluations (Real-time)

  • Above Expectation
    • case: when the learner crosses an entrance that the author defined Above Expectation assessment for
  • At Expectation
    • case: when the learner crosses an entrance that the author defined At Expectation assessment for
  • Below Expectation
    • case: when the learner crosses an entrance that the author defined Below Expectation assessment for
  • Unknown
    • case: the default value

Possible Scored Evaluations (overall)

  • count - the number of times the learner crosses an entrance that was authored with Below Expectation assessment.
  • time - none (because the condition never completes)

Miscellaneous details

  • NEVER completes, i.e. the learner can be assessed on all entrances any number of times
  • can be repeated

Intended Training Applications and Domains

  • any application that can cause GIFT Entity state messages to be created

Consumed State Messages (used for assessment)

  • GIFT Entity State
