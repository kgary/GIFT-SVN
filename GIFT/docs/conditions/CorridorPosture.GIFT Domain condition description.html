Check posture along path

This condition checks whether the learner has stays within the bounds of an entered path AND while in a path segment the learner maintains one of the specified postures (e.g. prone, kneeling, standing...).  This is useful for areas where you would like the learner to stay exposed on concealed within a certain pre-defined cylinder (e.g. stay low as you move along a short wall to prevent exposing yourself as a target or prevent reveling your movements to other characters in the application).  A path segment must have an entrance and exit as well as a width to the segmentt.  You may also provide a buffer value that is used at the edge along this segment to give the learner leeway when it comes to assessing whether the learner is still within the path.

Assessment Evaluations (Real-time)

  • Above Expectation
  • At Expectation
    • case: the learner entered a path segment, has not exited that segment, is still within that segment's boundary AND is currently in one of the defined postures for the path
    • case: the learner entered a path segment, has not exited that segment, went outside of that segment and has now come back into that segment's boundary AND is currently in one of the defined postures for the path
  • Below Expectation
    • case: the learner entered a path segment and before exiting that segment the learner went outside of that segment's boundary OR is currently NOT in one of the defined postures for that segment
  • Unknown
    • case: the default value

Possible Scored Evaluations (overall)

  • counts the total number of times the learner went outside of all segment's boundaries PLUS  the number of times the learner was not in the correct posture for a segment that the learner has entered but has not properly exited
  • provides the amount of time the learner was in violation of being outside of a path segment PLUS the amount of time not in the correct posture while inside the segment, i.e. the total amount of time this condition was at Below Expectation.

Miscellaneous details

  • NEVER completes, i.e. the learner can enter/exit and be assessed on all path segments any number of times
  • can be repeated

Intended Training Applications and Domains

  • any application that can cause GIFT Entity state messages to be created

Consumed State Messages (used for assessment)

  • GIFT Entity State
