Rules of Engagement Condition

This condition checks whether the learner has violated the rules of engagement.  To assess this weapon fire events caused by the learner are checked against the specified rules of engagement criteria.

Assessment Evaluations (Real-time)

    Default: 

  • Above Expectation
  • At Expectation
    • case: the default value
    • case: the weapon fire event is caused by the learner but the rules of engagement weapon control status is 'Free'
  • Below Expectation
    • case: the weapon fire event is caused by the learner but the rules of engagement weapon control status is 'Hold'
  • Unknown

    Customize:

  • count and completion time: an evaluation event happens each time the rules of engagement are violated.

Possible Scored Evaluations (overall)

  • counts the number of times the learner violates the weapon control status, i.e. number of times the condition assesses at Below Expectation

Miscellaneous details

  • the condition does not end
  • the condition can be repeated

Intended Training Applications and Domains

  • Any application that provides weapon fire information

Consumed State Messages (used for assessment)

  • GIFT Weapon fire
