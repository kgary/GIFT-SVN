Has Moved Excavator Component Condition

This condition checks whether the learner has moved a specific Excavator component (e.g. boom, bucket) enough to warrant an assessment.  The author must provide one or more components that could be moved for this assessment.  For each component the type of direction allowed {any direction, bidirectional} is optional (the default being bidirectional).

The expected movements for each component are as follows:

Swing - 90 degrees, Boom - 90 degrees, Arm - 180 degrees, Bucket, 180 degrees

Assessment Evaluations (Real-time)

  • Above Expectation
    • case: the correct component is moved the desired distance as defined by the author
  • At Expectation
  • Below Expectation
    • case: the default value
    • case: the wrong component is moved, i.e. another component besides the one this condition has been configured to assess is being moved
  • Unknown
    • case: the default value

Possible Scored Evaluations (overall)

  • counts the number of times the movement assessment is assessed (positvely or negatively, right or wrong)
  • provides the amount of time it took the condition to complete, i.e. the learner has completed all assessments by moving the specified component

Miscellaneous details

  • the condition completes once the excavator component movement is assessed the maximum number of assessment times as defined by the author
  • the condition can't be repeated
