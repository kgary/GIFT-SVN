Follow Path Condition

This condition checks whether the learner has stays within the bounds of an entered path.  This is useful for areas where you would like the learner to stay within a certain pre-defined cylinder.  A path segment must have an entrance and exit as well as a width to the segment.  You may also provide a buffer value that is used at the edge along this segment to give the learner leeway when it comes to assessing whether the learner is still within the path.

Assessment Evaluations (Real-time)

  • Above Expectation
  • At Expectation
    • case: the learner entered a path segment, has not exited that segment and is still within that segment's boundary
    • case: the learner entered a path segment, has not exited that segment, went outside of that segment and has now come back into that segment's boundary
  • Below Expectation
    • case: the learner entered a path segment and before exiting that segment the learner went outside of that segment's boundary
  • Unknown
    • case: the default value

Possible Scored Evaluations (overall)

  • counts the total number of times the learner went outside of all segment's boundaries for a path segment that the learner has entered but has not properly exited
  • provides the amount of time the learner was in violation of being outside of a path segment, i.e. the total amount of time this condition was at Below Expectation.

Miscellaneous details

  • NEVER completes, i.e. the learner can enter/exit and be assessed on all path segments any number of times
  • can be repeated

Intended Training Applications and Domains

  • any application that can cause GIFT Entity state messages to be created

Consumed State Messages (used for assessment)

  • GIFT Entity State
