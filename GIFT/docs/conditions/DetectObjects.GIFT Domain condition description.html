Detect Objects Condition

This condition assesses whether one or more team members are properly detecting objects within a window of time after the object has entered the field of view.

Currently, to detect an object means to orient to that object within a specified cone in degrees after that object has entered the larger field of view.  In the future other means of 'detection' might be supported such as pressing a button, verbal communication or other action naitve to the training environment.

The author can define the objects, time to detect, field of view, and orient cone as inputs to the assessment algorithm.  Only objects that are healthy will be considered for assessment.  This prevents being assessed against objects that are dead/incapacitated. 

The goal of this condition is to not assess every possible situation.  Hopefully it provides an initial assessment result that can be used by observers/trainers to make additional assessments or over-rides.

Assessment Evaluations (Real-time)

  • Above Expectation
    • case: assessed team member(s) have detected objects within the time specificed to receive an Above Expectation. 
    • case: the average assessment of all detections in this instance averages to Above Expectation.
  • At Expectation
    • case: assessed team member(s) have detected objects within the time specified to receive an At Expectation. 
    • case: the average assessment of all detections in this instance averages to At Expectation.
  • Below Expectation
    • case: assessed team member(s) have detected objects beyond the time specified to receive an At or Above Expectation.  This includes not detecting objects that come within the field of view.
    • case the average assessment of all detections in this instance averages to Below Expectation.
  • Unknown
    • case: The default initial value of this condition.

Possible Scored Evaluations (overall)

  • counts the number of violations (going from At/Above to Below Expecation)
  • sums the duration of time in Below Expectation

Miscellaneous details

  • this condition DOES complete when all objects have been assessed (i.e. found within the field of view and either detected or not detected)

Intended Training Applications and Domains

  • any application that can produce entity state

Consumed State Messages (used for assessment)

  • GIFT Entity State
