Has Collided Condition

This condition checks whether the learner has collided with another object by analyzing collision messages.

Currently if any collision message is received it will cause an assessment event, even if the collision was not caused by the learner.

Assessment Evaluations (Real-time)

  • Above Expectation
  • At Expectation
    • case: each time a collision message is received (ignoring multiple collision events within a 1 second window of the first collision event)
  • Below Expectation
  • Unknown
    • case: the default value

Possible Scored Evaluations (overall)

  • count - the number of collisions
  • time - not used (because the condition does not complete)

Miscellaneous details

  • the condition does not complete
  • the condition can be repeated

Intended Training Applications and Domains

  • applications that use the DIS GIFT Gateway interop plugin
  • applications that can produce the GIFT Collision message

Consumed State Messages (used for assessment)

  • GIFT Collision
