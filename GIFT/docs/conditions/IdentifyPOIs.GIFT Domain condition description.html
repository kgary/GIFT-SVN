Identify Points of Interest (POIs) Condition

This condition checks whether the learner has seen a set of points.  The set of points is provided as input to the condition.

Currently to identify a point means that the underyling application must provide information about whether the learner can see that location.  This is performed by having this condition class send a request to the Gateway module which than should query the application for an answer on a particular set of points.  In the case of Virtual Battlespace (VBS), this is answered by performing Line of sight (LoS) queries in VBS.  For performance reasons, this condition class will only request LoS queries once every second until all points have been seen.

Assessment Evaluations (Real-time)

    Default: 

  • Above Expectation
  • At Expectation
    • case: the learner has seen all points
  • Below Expectation
    • case: the default value
    • case: until all points are seen
  • Unknown

    Customize:

  • count: an evaluation event happens once for point in the authored list when the point is identified.  The number of events will be less than or equal to the number of points.
  • completion time: amount of time it takes the learner to identify all of the points in the list.

Possible Scored Evaluations (overall)

  • counts the number of points that were seen
  • provides the time it took for the learner to see all the points

Miscellaneous details

  • the condition completes once all points have been seen as reported by the external application
  • can not be repeated

Intended Training Applications and Domains

  • Virtual Battlespace (VBS)
  • any application that can send entity state and respond to GIFT LOS requests with a GIFT LOS result

Consumed State Messages (used for assessment)

  • GIFT LOS Result
  • GIFT Entity State
