Speed Limit Condition

This condition checks whether the object/player controlled by the learner is violating a speed limit.  The condition must be configured with a speed limit (mph). 

YOU MUST ASSIGN AN ENTITY TO THE LEARNER IN THIS DKF FOR THIS CONDITION TO WORK.

Assessment Evaluations (Real-time)

    Default: 

  • Above Expectation
  • At Expectation
    • case: the speed is below the speed limit (one At Expectation assessment will occur for each change in state from violating to not violating the condition)
    • case: the default, starting value for this condition
  • Below Expectation
    • case: the current speed is greater than the speed limit (one Below Expectation assessment will occur for each change in state from not violating to violating the condition)
  • Unknown

    Customize:

  • count : an evaluation event happens each time the learner is above the speed limit. 
  • time : amount of time the learner is violating the speed limit.

Possible Scored Evaluations (overall)

  • counts the number of violations (going from At to Below Expecation)
  • sums the duration of time in Below Expectation

Miscellaneous details

  • the condition never completes
  • YOU MUST ASSIGN AN ENTITY TO THE LEARNER IN THIS DKF FOR THIS CONDITION TO WORK.
  • for each change in violation of the condition only a single assessment is sent (see Assessment Evaluations above)

Intended Training Applications and Domains

  • any application that can cause GIFT Entity state messages to be created
  • GIFT Mobile app

Consumed State Messages (used for assessment)

  • GIFT Entity State
  • GIFT Geolocation
