Observed Assessment Condition

This condition is used to indicate that the concept will not involve GIFT automatically assessing it and will instead be monitored manually by an instructor who will use the Game Master application to provide real time assessment for this concept. For example: the instructor might decide that a formation is performing below expectation and push that value to the concept.

Assessment Evaluations (Real-time)

  • none

Possible Scored Evaluations (overall)

  • none

Miscellaneous details

  • NEVER completes, i.e. the instructor can continuously monitor and update the concept

Intended Training Applications and Domains

  • any application that is monitored by an instructor instead of GIFT automatically assessing the concept

Consumed State Messages (used for assessment)

  • GIFT Entity State - this is to satisfy a validation check on the condition class, these messages are not used in this condition.

GIFT class: mil.arl.gift.domain.knowledge.condition.ObservedAssessmentCondition
