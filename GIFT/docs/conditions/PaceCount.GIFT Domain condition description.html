Pace Count Condition

This condition checks whether the object/player controlled by the learner has moved a specified distance.  The learner must indicate the starting and stoping point for this distance check by using the Start and Stop pace count learner actions accordingly.  The condition must be configured with an ideal distance in meters.  In addition a threshold in meters can be authored to provide flexibility on how exact the distance traveled must be to the ideal distance. 

YOU MUST ASSIGN AN ENTITY TO THE LEARNER IN THIS DKF FOR THIS CONDITION TO WORK. [Except when using a mobile device and the mobile events course object]

Assessment Evaluations (Real-time)

  • Above Expectation
    • case: the distance traveled from when the start pace count learner action button was pressed until the end pace count learner action button was pressed is within the ideal distance authored +/- the threshold distance.
  • At Expectation
    • case: the distance traveled from when the start pace count learner action button was pressed to the current learner location is within the ideal distance authored +/- the threshold distance.  This is useful for debugging purposes as an instructional strategy (e.g. feedback) can be shown to indicate that the correct distance has been reached.  This At Expectation value will be sent for every location update while the learner is within the threshold distance from the ideal distance traveled.
  • Below Expectation
    • case: the distance traveled from when the start pace count learner action button was pressed until the end pace count learner action button was pressed is outside of the ideal distance authored +/- the threshold distance.
  • Unknown
    • case: the default, starting value for this condition

Possible Scored Evaluations (overall)

  • the total amount of time between pressing the start and stop pace count learner action buttons

Miscellaneous details

  • the condition completes when the stop pace count learner action button is pressed
  • YOU MUST ASSIGN AN ENTITY TO THE LEARNER IN THIS DKF FOR THIS CONDITION TO WORK.  [Except when using a mobile device and the mobile events course object]

Intended Training Applications and Domains

  • any application that can cause GIFT Entity state or Geo Location messages to be created (e.g. Virtual BattleSpace (VBS) and GIFT mobile app, respectively)

Consumed State Messages (used for assessment)

  • GIFT Entity State
  • Geo Location
  • Learner Tutor Action
