Checkpoint Progress Condition

This condition checks the pace of the learner's player/character/entity in the external application through one or more, ordered, checkpoints.  The condition requires one or more checkpoints to be provided, each defined by a location (e.g. GDC) with a time constraint.  Each time constraint can also have a window of opportunity associated with it to act as a grace period for the learner to reach that checkpoint and not be negatively assessed.

This is different than Checkpoint Pace condition because in this condition a timer is created for each checkpoint using the elapsed scenario time defined in the checkpoint's time constraint.  If a checkpoint's timer reaches zero and the learner has not arrived at that checkpoint's location, the learner will be negatively assessed.

Assessment Evaluations (Real-time)

  • Above Expectation
    • case: the learner reached the next checkpoint in the list ahead of schedule
  • At Expectation
    • case: reached the next checkpoint in the list within the time constraint grace period (window)
  • Below Expectation
    • case: a checkpoint's timer reached zero and the learner has not arrived at that checpoint's location
  • Unknown
    • case: the default value

Possible Scored Evaluations (overall)

  • counts the number of checkpoints that were not reached on time
  • provides the amount of time the learner was in violation of being behind progress, i.e. the total amount of time this condition was at Below Expectation.

Miscellaneous details

  • completes after the last checkpoint location has been reached and assessed
  • can not be repeated, i.e. once you reach the last checkpoint you can't start this same condition instance from the first checkpoint

Intended Training Applications and Domains

  • any application that can cause GIFT Entity state messages to be created
  • GIFT Mobile App

Consumed State Messages (used for assessment)

  • GIFT Entity State
  • GIFT Geolocation
