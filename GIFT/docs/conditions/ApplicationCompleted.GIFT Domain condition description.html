Application Completed Condition

This condition checks whether the currently running application (e.g. VBS, PowerPoint, Unity) has stopped or finished.  The external application communicates this by sending on the supported simulation management message (SIMAN) listed below.  Completing this condition is a useful way to determine when GIFT performance assessments should end.  The condition can provide a single assessment based on the ideal amount of time it should take to finish the application.

Assessment Evaluations (Real-time)

  • Above Expectation
  • At Expectation
    • case: a Siman.Stop_Freeze message is received and that message represents a stop and not a freeze/pause.
  • Below Expectation
    • case: the amount of time defined in the optional ideal completion duration parameter has elapsed and the external application is still running.  This condition will not include the amount of time the application was paused against this idea completion duration as long as the application provided the appropriate Siman.Stop_Freeze message to GIFT indicating when the pause took place.
  • Unknown

Possible Scored Evaluations (overall)

  • counts the number of times this condition completes, i.e. the number of times the condition receives the Siman.Stop_Freeze and is not a freeze/pause notification.

Miscellaneous details

  • the condition completes when a Siman.Stop_Freeze messsage is received and it is not a freeze/pause.
  • the condition is not meant to be repeated

Intended Training Applications and Domains

  • any application that can produce Siman messages

Consumed State Messages (used for assessment)

  • GIFT Start Resume
  • GIFT Stop Freeze

GIFT class: mil.arl.gift.domain.knowledge.condition.ApplicationCompletedCondition
