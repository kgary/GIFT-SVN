Spacing Condition

This condition checks whether two objects/players controlled by learners are maintaining proper spacing (or formation).  The condition must be configured one or more pairs of spacing information.

YOU MUST ASSIGN AN ENTITY TO THE LEARNER IN THIS DKF FOR THIS CONDITION TO WORK.

Assessment Evaluations (Real-time)

    Default: 

  • Above Expectation
    • case: all of the spacing pairs are within the ideal spacing distance range
  • At Expectation
    • case: all of the spacing pairs are within the accepted spacing distance range (some could be within the ideal spacing distance range but at least one pair is not)
    • case: the default, starting value for this condition
  • Below Expectation
    • case: at least one of the spacing pairs is outside of the accepted (and ideal) spacing distance ranges.
  • Unknown

    Customize:

  • count : an evaluation event happens each time there is a transition from At/Above to Below Expectation,
  • time : amount of time the condition is being violated.

Possible Scored Evaluations (overall)

  • counts the number of violations (going from At/Above to Below Expecation)
  • sums the duration of time in Below Expectation

Miscellaneous details

  • the condition never completes
  • YOU MUST ASSIGN AN ENTITY TO THE LEARNER IN THIS DKF FOR THIS CONDITION TO WORK.
  • for each change in violation of the condition only a single assessment is sent (see Assessment Evaluations above)

Intended Training Applications and Domains

  • any application that can cause GIFT Entity state messages to be created

Consumed State Messages (used for assessment)

  • GIFT Entity State
