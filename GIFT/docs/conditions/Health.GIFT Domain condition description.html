Health Condition

This condition checks whether team members in the team organization are not healthy, i.e. have some type of damage state.

Assessment Evaluations (Real-time)

    Default: 

  • Above Expectation
  • At Expectation
    • case: the default, starting value for this condition
    • case: all of the referenced team members are healthy (one At Expectation assessment will occur for each change in state from violating to not violating the condition)
  • Below Expectation
    • case: one or more of the referenced team members are not healthy (one Below Expectation assessment will occur for each change in state from not violating to violating the condition)

    Customize:

  • count : an evaluation event happens each time the condition goes from At to Below expectation.
  • time : amount of time the condition is at Below Expectation.  Time can be used to assess how long the referenced team members were not healthy.

Possible Scored Evaluations (overall)

  • counts the number of violations (going from At to Below Expecation)
  • sums the duration of time in Below Expectation

Miscellaneous details

  • the condition never completes
  • for each change in violation of the condition only a single assessment is sent (see Assessment Evaluations above)

Intended Training Applications and Domains

  • any application that can cause GIFT Entity state messages to be created

Consumed State Messages (used for assessment)

  • GIFT Entity State
