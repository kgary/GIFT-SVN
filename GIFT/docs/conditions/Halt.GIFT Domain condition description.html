Halt Condition

This condition checks whether the objects/players controlled by the learner stop moving after the first learner stops moving.  This condition can be configured with a custom assessment that will provide assessment based on how long it takes the learners to halt.

YOU MUST ASSIGN AN ENTITY TO THE LEARNER IN THIS DKF FOR THIS CONDITION TO WORK.

Assessment Evaluations (Real-time)

    Default: 

  • Above Expectation
  • At Expectation
    • case: all of the referenced learners have stopped moving (one At Expectation assessment will occur for each change in state from violating to not violating the condition)
  • Below Expectation
    • case: one or more of the referenced learners are moving (one Below Expectation assessment will occur for each change in state from not violating to violating the condition)
  • Unknown
    • case: the default, starting value for this condition

    Customize:

  • count : an evaluation event happens each time the condition goes from At to Below expectation.
  • time : amount of time the condition is at Below Expectation.  Time can be used to assess how long the halt takes starting from when the first learner stops moving until when the last reference learner stops moving.

Possible Scored Evaluations (overall)

  • counts the number of violations (going from At to Below Expecation)
  • sums the duration of time in Below Expectation

Miscellaneous details

  • the condition never completes
  • YOU MUST ASSIGN AN ENTITY TO THE LEARNER IN THIS DKF FOR THIS CONDITION TO WORK.
  • for each change in violation of the condition only a single assessment is sent (see Assessment Evaluations above)

Intended Training Applications and Domains

  • any application that can cause GIFT Entity state messages to be created

Consumed State Messages (used for assessment)

  • GIFT Entity State
